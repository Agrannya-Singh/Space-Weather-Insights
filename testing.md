# Space Weather Insights: E2E Testing Documentation

This document outlines the End-to-End (E2E) testing methodology, coverage, and results for the Space Weather Insights dashboard.

## Methodology

We use **Playwright** for automated E2E testing. The tests are designed to verify the core user journeys and UI integrity across the deployed application.

### Key Principles
1.  **Deployment-Centric**: Tests run directly against the live URL (`https://space-weather-insights.vercel.app/`).
2.  **Cross-Browser Readiness**: Configured for Chromium (Chrome/Edge), Firefox, and WebKit (Safari).
3.  **Resilient Locators**: Emphasis on user-facing attributes like text content and roles (using Regex for flexibility).
4.  **Modular Test Suites**: Separated by feature area (Homepage, EDA, Explanation, Navigation).

## How to Run Tests Locally

```bash
# Install dependencies
npm install

# Install browsers (if not already present)
npx playwright install chromium

# Run all tests
npm run test:e2e

# Run with visible UI
npm run test:e2e:headed

# View the latest test report
npm run test:e2e:report
```
